# XRF55 CSI-HAR Benchmark

Benchmark of WiFi-CSI **human activity recognition (HAR)** models on the
[XRF55](https://aiotgroup.github.io/XRF55/) dataset, with a unified trainer, three
reproducible training protocols, and a custom wavelet–Mamba model (**WavDualMamba**).

| Model | Description |
|---|---|
| `resnet`          | ResNet18-1D over the 270-channel CSI amplitude |
| `tfmamba`         | TF-Mamba — two-stream (cH / cV) Mamba |
| `wavmamba`        | WavMambaHAR — DWT subbands + Mamba |
| `wavdualmamba`    | Multi-branch (per-DWT-subband) CNN + gated BiMamba + adaptive late fusion |
| `wavdualmamba_v2` | WavDualMamba variant |

## Dataset

XRF55 amplitude subset — **11 activities**, **30 subjects**, **20 repetitions**.

- Split: **train = reps 1–14 (4620 samples)**, **test = reps 15–20 (1980 samples)**. No val.
- Classes: Waving, Clap Hands, Fall on the Floor, Jumping, Running, Sitting Down,
  Standing Up, Turning, Walking, Stretch Oneself, Pat on Shoulder.
- Prebuilt arrays: [`xrf55-amp-dataset`](https://www.kaggle.com/datasets/imhoangt/xrf55-amp-dataset)
  (Kaggle mount: `/kaggle/input/datasets/imhoangt/xrf55-amp-dataset/`).

## Installation

```bash
git clone https://github.com/imhoangt/xrf55_benchmark.git
cd xrf55_benchmark
pip install -e .                 # core deps + the xrf55_bench package
```

Mamba models additionally need a **CUDA GPU** and a special install order:

```bash
pip install ninja packaging wheel triton
pip install "causal-conv1d>=1.2.0" --no-build-isolation
pip install mamba-ssm            --no-build-isolation
```

## Quickstart

**Data pipeline** (run from repo root, in order):

```bash
python xrf55_bench/scripts/00_parse_csi.py             # raw .dat → complex npy
python xrf55_bench/scripts/01_build_dataset_raw.py     # raw amplitude bench arrays
python xrf55_bench/scripts/02_build_dataset_processed.py  # Hampel + Butterworth LPF
python xrf55_bench/scripts/03_plot_amplitude.py        # → assets/figures/
```

**Train**:

```python
from xrf55_bench.config  import TrainCfg_for_protocol
from xrf55_bench.trainer import run

cfg = TrainCfg_for_protocol('03', seeds=(42,))
run(model_name='wavdualmamba', bench_dir=BENCH_DIR, output_dir=OUTPUT_DIR, train_cfg=cfg)
```

See [`docs/protocols.md`](docs/protocols.md) for the three protocols and the per-run
output layout. Kaggle notebooks live in [`xrf55_bench/notebooks/`](xrf55_bench/notebooks/).

## Results

Headline result is the **final-epoch** metric (`last_model.pt`), not best-by-test.
`best_epoch` peeks at the test set during training and is a **diagnostic only**.

| Model | Protocol | Data | Test acc (final) | F1-macro | Params | Size | Latency (T4) |
|---|---|---|---|---|---|---|---|
| WavDualMamba (LL/HL/LH, pool_ctx, no ECA) | 03 | processed | **92.83%** | 92.71% | 0.588 M | 2.44 MB | 13.11 ms |

Single seed (42). Multi-seed runs are recommended before reporting in a paper.
